# TodoリストWebアプリ 要件定義書

> スコープ制御方針: [.kiro/steering/scope.md](../../steering/scope.md) に従う。要件にない機能は追加しない。

## 1. システム概要

本システムは、利用者が日常的なタスクを管理するためのWebアプリケーションである。
利用者は、ブラウザ上でタスクの追加・一覧表示・完了状態の変更・削除を行うことができる。
ログイン機能やユーザー管理機能は対象外とし、1人の利用者がローカル環境で使うことを前提とする。

## 2. 利用環境

- ブラウザで利用する
- ローカル環境（自分のPC）で動作する
- アクセスURLは `http://localhost:5001/` とする（macOSではポート5000をAirPlayが使うため5001を使用）

## 3. 機能要件

### 3.1 タスク一覧表示

- 登録済みのタスクを一覧表示する
- 未完了タスクと完了済みタスクを分けて表示する

### 3.2 タスク追加

- 利用者はタスク名を入力してタスクを追加できる
- タスク名が未入力（空文字や空白のみ）の場合は追加せず、メッセージを表示する

### 3.3 タスク完了状態の変更

- 利用者は未完了タスクを完了済みに変更できる
- 利用者は完了済みタスクを未完了に戻せる
- 完了済みタスクは見た目で区別して表示する

### 3.4 タスク削除

- 利用者は任意のタスクを削除できる

## 4. データ管理

- タスクデータはJSONファイル（`data/todos.json`）に保存する
- ブラウザを閉じても、アプリを再起動してもデータを保持する
- データベース（DBMS）は使用しない

保存データの例:

```json
[
  { "id": 1, "title": "買い物に行く", "completed": false },
  { "id": 2, "title": "宿題をやる", "completed": true }
]
```

データ項目:

- `id`: タスクを識別する番号（整数、重複しない）
- `title`: タスク名（文字列）
- `completed`: 完了状態（`true` = 完了済み、`false` = 未完了）

## 5. 技術要件・構成要件

- Python / Flask を利用する
- 画面は HTML / CSS / JavaScript で作る
- 仮想環境は venv を利用する
- `requirements.txt` を用意する
- ファイル読み書きなどでエラーが起きても、アプリが異常終了しないようにする
- アプリ本体・テンプレート・静的ファイル・データファイルを分けた構成にする
- 以下のファイル構成で作成する

```text
todo-app/
├── app.py
├── data/
│   └── todos.json
├── templates/
│   └── index.html
├── static/
│   ├── css/
│   │   └── style.css
│   └── js/
│       └── main.js
├── requirements.txt
└── README.md
```

## 6. 対象外（作らない機能）

- ログイン機能
- ユーザー管理機能
- データベース（DBMS）によるデータ管理
- タスクの期限
- 優先度
- 検索
- 並び替え

上記に加え、要件に書かれていない機能は一切追加しない。
